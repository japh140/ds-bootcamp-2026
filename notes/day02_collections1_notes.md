# Day 2 — Collections I Notes

## Main idea
Collections let you store more than one value in a single variable.

## Types covered

### String
- A string stores text
- Strings are ordered
- You can index and slice them
- Example: "python"

### List
- A list stores multiple values
- Lists are ordered
- Lists can be changed (mutable)
- Example: [1, 2, 3]

### Tuple
- A tuple also stores multiple values
- Tuples are ordered
- Tuples cannot be changed after creation (immutable)
- Example: (1, 2, 3)

## Key concepts

### Indexing
- Indexing means accessing one item by position
- Python starts counting from 0
- Example:
  - text[0] gives the first character
  - nums[2] gives the third item

### Negative indexing
- -1 means the last item
- -2 means the second last item

### Slicing
- Slicing means taking part of a collection
- Example:
  - text[0:3]
  - items[1:4]

### List methods
- append() adds one item to the end
- remove() removes a matching value
- pop() removes by position, default last item

### Useful built-ins
- len() gives the number of items
- type() shows the type
- sorted() returns a sorted version of a collection
- in checks membership

## Important differences
- Strings: text
- Lists: changeable collection
- Tuples: fixed collection

## Common mistakes
- Forgetting Python starts at index 0
- Going out of range with an index
- Mixing up append() and remove()
- Forgetting tuples cannot be edited
- Confusing slicing end position as inclusive
